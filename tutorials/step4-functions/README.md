## 📚 핵심 개념

### 함수 정의와 호출
```freelang
함수 함수명(매개변수1, 매개변수2) {
  // 함수 본문
  반환 결과
}

// 호출
함수명(인자1, 인자2)
```

### 매개변수와 인자
- **매개변수**: 함수 정의 시 입력받을 변수
- **인자**: 함수 호출 시 전달하는 실제 값

### 반환값
함수에서 결과를 돌려주는 값

```freelang
함수 두배(숫자) {
  반환 숫자 * 2
}

변수 결과 = 두배(5)  // 결과는 10
```

### 재귀 함수
함수 자신을 호출하는 함수

```freelang
함수 팩토리얼(n) {
  만약 n <= 1 {
    반환 1
  }
  반환 n * 팩토리얼(n - 1)
}
```

## 📝 예제 코드

### 예제 1: 기본 함수
파일: `1-basic-functions.free`

```freelang
// 간단한 함수
함수 인사() {
  출력("안녕하세요!")
}

// 호출
인사()

// 반환값이 있는 함수
함수 더하기(a, b) {
  반환 a + b
}

변수 합 = 더하기(5, 3)
출력("5 + 3 = " + 합)
```

### 예제 2: 매개변수와 반환값
파일: `2-parameters.free`

```freelang
함수 섭씨_화씨(섭씨) {
  반환 (섭씨 * 9 / 5) + 32
}

함수 원_달러(원) {
  반환 원 / 1300
}

변수 f = 섭씨_화씨(25)
출력("25°C = " + f + "°F")

변수 달러 = 원_달러(13000)
출력("13000원 = " + 달러 + "달러")
```

### 예제 3: 재귀 함수
파일: `3-recursion.free`

```freelang
함수 팩토리얼(n) {
  만약 n <= 1 {
    반환 1
  }
  반환 n * 팩토리얼(n - 1)
}

함수 피보나치(n) {
  만약 n <= 1 {
    반환 n
  }
  반환 피보나치(n - 1) + 피보나치(n - 2)
}

출력("5! = " + 팩토리얼(5))
출력("피보나치(10) = " + 피보나치(10))
```

### 문제 1: 면적과 둘레 계산
원의 면적과 둘레를 계산하는 함수를 작성하세요.
반지름이 5인 원의 면적과 둘레를 계산하세요.

**힌트**:
- 원의 면적 = π × r²
- 원의 둘레 = 2 × π × r
- π는 3.14로 계산하세요

### 문제 2: 절댓값 구하기

```freelang
함수 절댓값(숫자) {

출력(절댓값(-10))  // 10
출력(절댓값(5))    // 5
```

## 💾 솔루션

### 문제 1 솔루션
```freelang
함수 원_면적(반지름) {
  반환 3.14 * 반지름 * 반지름
}

함수 원_둘레(반지름) {
  반환 2 * 3.14 * 반지름
}

변수 면적 = 원_면적(5)
변수 둘레 = 원_둘레(5)

출력("면적: " + 면적)
출력("둘레: " + 둘레)
```

### 문제 2 솔루션
```freelang
함수 절댓값(숫자) {
  만약 숫자 < 0 {
    반환 숫자 * -1
  }
  반환 숫자
}

출력(절댓값(-10))
출력(절댓값(5))
```

### 문제 3 솔루션
```freelang
함수 최대(a, b, c) {
  변수 최대값 = a

  만약 b > 최대값 {
    최대값 = b
  }

  만약 c > 최대값 {
    최대값 = c
  }

  반환 최대값
}

출력(최대(10, 20, 5))
```

## 🔑 중요한 규칙

```freelang
// 좋은 함수명
함수 학생_이름_검증(이름) { }
함수 계산_최대값(a, b) { }

// 한글과 영문 혼용 가능
함수 calculateSum(a, b) { }
```

```freelang
함수 테스트() {
  출력("테스트")
  // 명시적 반환 없으면 특수값 반환
```

## 💡 팁

### 함수를 사용하는 이유
1. **재사용성**: 같은 코드를 여러 번 사용 가능
2. **가독성**: 코드를 더 이해하기 쉽게 만듦
3. **유지보수**: 한 곳만 수정하면 모든 곳에 반영

### 재귀 함수 작성 팁
- 반드시 **종료 조건**이 있어야 함
- 무한 재귀를 피할 것
- 깊은 재귀는 성능 문제 가능
