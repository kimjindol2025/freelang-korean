## 성능 프로파일링

### 1. 런타임 프로파일링

K-FreeLang은 빌트인 프로파일러를 제공합니다.

```freelang
// 프로파일 활성화
프로파일_시작()

// 성능 측정 코드
함수 느린함수() → 공집합 {
  반복 변수 i = 0; i < 1000000; i = i + 1 {
    변수 x = i * 2
    출력(x)
  }
}

느린함수()

// 결과 출력
프로파일_결과_출력()
```

**예상 출력:**
```
함수: 느린함수
  총 시간: 234ms
  호출 횟수: 1
  평균: 234ms
  자식 호출: 출력(1000000회)
```

### 2. 메모리 프로파일링

메모리 사용 패턴을 분석합니다.

```freelang
메모리프로파일_시작()

// 대량 할당
변수 큰배열 = 배열생성(1000000)
반복 변수 i = 0; i < 1000000; i = i + 1 {
  큰배열[i] = i
}

메모리프로파일_결과() // 힙 사용량 출력
```

### 3. IR 최적화 분석

컴파일 단계에서 최적화 결과를 확인합니다.

```bash
k-freelang compile myprogram.free --optimize --verbose
# 출력:
# [Optimization Pass] Constant Folding: 10 → 3 instructions
# [Optimization Pass] Dead Code Elimination: 3 → 2 blocks
# [Optimization Pass] Label Optimization: 5 → 2 labels
```

## 상수 폴딩 최적화

### 1. 컴파일 타임 계산

가능한 한 상수 표현식을 미리 계산합니다.

```freelang
// ❌ 비효율적: 런타임 계산
함수 계산() → 숫자 {
  변수 결과 = 10 + 20 + 30 + 40  // 매번 실행 시 계산
  반환 결과
}

// ✅ 최적화: 컴파일 타임 계산
함수 계산() → 숫자 {
  상수 결과 = 10 + 20 + 30 + 40  // 컴파일 타임에 100으로 폴딩
  반환 결과
}
```

**최적화 결과:**
```
원본 IR:  LoadConst(10) → LoadConst(20) → Add → ... (4개 명령어)
최적화:   LoadConst(100) (1개 명령어)
성능:     75% 감소
```

### 2. 반복 중 불변식 추출 (Loop-Invariant Code Motion)

루프 바깥으로 불변 계산을 이동합니다.

```freelang
// ❌ 비효율적: 루프 내에서 반복 계산
함수 합계계산(배열: 배열<숫자>) → 숫자 {
  변수 합 = 0
  상수 배수 = 2

  반복 변수 i = 0; i < 길이(배열); i = i + 1 {
    변수 스케일 = 배수 * 10  // 매번 계산 (불변)
    합 = 합 + (배열[i] * 스케일)
  }

  반환 합
}

// ✅ 최적화: 불변식을 루프 밖으로
함수 합계계산(배열: 배열<숫자>) → 숫자 {
  변수 합 = 0
  상수 배수 = 2
  상수 스케일 = 배수 * 10  // 루프 밖에서 계산

  반복 변수 i = 0; i < 길이(배열); i = i + 1 {
    합 = 합 + (배열[i] * 스케일)
  }

  반환 합
}
```

**성능 개선:**
- 배열 크기 1,000: 9,000회 불필요한 계산 제거
- 배열 크기 1,000,000: 9,999,000회 계산 제거

### 3. 강도 감소 (Strength Reduction)

비용이 높은 연산을 저비용 연산으로 변환합니다.

```freelang
// ❌ 비효율적: 곱셈 (느림)
반복 변수 i = 0; i < 100; i = i + 1 {
  변수 오프셋 = i * 8  // 곱셈 (느림)
}

// ✅ 최적화: 덧셈 (빠름)
변수 오프셋 = 0
반복 변수 i = 0; i < 100; i = i + 1 {
  // 오프셋은 매 반복 +8 증가
  오프셋 = 오프셋 + 8  // 덧셈 (빠름)
}
```

**성능 개선:**
- 현대 CPU: 곱셈 ~3 사이클, 덧셈 1 사이클
- 100회 반복: 200 사이클 절감

## 데드 코드 제거

### 1. 도달 불가능 코드

조건부 분기 후 도달 불가능한 코드를 제거합니다.

```freelang
// ❌ 비효율적: 도달 불가능한 코드
함수 예제() → 숫자 {
  반환 42

  출력("이 코드는 실행되지 않음")  // 도달 불가능 (제거됨)
  변수 x = 10                        // 도달 불가능 (제거됨)
  반환 100                           // 도달 불가능 (제거됨)
}

// ✅ 최적화 후:
함수 예제() → 숫자 {
  반환 42
}
```

### 2. 사용되지 않는 변수

선언되지만 사용되지 않는 변수를 제거합니다.

```freelang
// ❌ 미사용 변수
함수 계산(a: 숫자, b: 숫자) → 숫자 {
  변수 중간값 = a + b  // 선언되었지만 미사용
  변수 임시 = 10        // 미사용

  반환 a * b
}

// 경고:
// Warning: Variable 'intermediateValue' declared but never used
// Warning: Variable 'temp' declared but never used
```

### 3. 조건부 분기 단순화

항상 참/거짓인 조건을 단순화합니다.

```freelang
상수 DEBUG = 거짓

// ❌ 비효율적:
만약 DEBUG {
  디버그로깅()  // 항상 실행 안 됨
}

// ✅ 최적화: 전체 if 블록 제거
// (DEBUG가 컴파일 타임 상수이므로)
```

## 메모리 효율성

### 1. 메모리 풀 활용

빈번하게 할당/해제되는 객체를 풀에서 재사용합니다.

```freelang
// 메모리 풀 생성
변수 풀 = 메모리풀_생성(크기: 1000)

// 재사용 가능한 객체 할당
함수 객체_획득() → 객체 {
  반환 풀.획득()
}

// 사용 후 반환
함수 객체_반환(obj: 객체) → 공집합 {
  풀.반환(obj)
}

// 예제: 1000개 요청 처리
반복 변수 i = 0; i < 1000; i = i + 1 {
  변수 요청 = 객체_획득()
  처리(요청)
  객체_반환(요청)  // 재사용을 위해 반환
}
```

### 2. 구조체 레이아웃 최적화

필드 순서를 메모리 패킹을 고려하여 배열합니다.

```freelang
// ❌ 비효율적: 패딩 낭비 많음
구조체 User_Bad {
  이름: 문자열,      // 8 바이트 (포인터)
  나이: 숫자,        // 4 바이트 (int32)
  // 4 바이트 패딩
  활성: 참거짓,      // 1 바이트 (bool)
  // 7 바이트 패딩
}
// 총 32 바이트

// ✅ 최적화: 패딩 최소화
구조체 User_Good {
  이름: 문자열,      // 8 바이트
  나이: 숫자,        // 4 바이트
  활성: 참거짓,      // 1 바이트
  // 3 바이트 패딩
}
// 총 16 바이트 (50% 절감)
```

### 3. 캐시 지역성

메모리 접근 패턴을 개선하여 캐시 히트율을 높입니다.

```freelang
// ❌ 비효율적: 무작위 접근
변수 배열 = 배열생성(1000000)
변수 스텝 = 13  // 소수 (캐시 라인 경합)

반복 변수 i = 0; i < 1000000; i = i + 스텝 {
  배열[i] = i  // 캐시 미스 높음
}

// ✅ 최적화: 순차 접근
반복 변수 i = 0; i < 1000000; i = i + 1 {
  배열[i] = i  // 캐시 히트율 높음
}
```

## 동시성 성능

### 1. Lock Contention 최소화

뮤텍스 보유 시간을 최소화합니다.

```freelang
// ❌ 비효율적: 장시간 lock 보유
동기화 {
  변수 데이터 = 비용많은계산()  // 시간 소요
  공유자원에저장(데이터)         // 짧은 작업
}

// ✅ 최적화: lock 범위 축소
변수 데이터 = 비용많은계산()    // lock 없이
동기화 {
  공유자원에저장(데이터)        // 필요한 부분만 lock
}
```

### 2. Fine-Grained Locking

더 세분화된 lock으로 병렬성을 높입니다.

```freelang
// ❌ 비효율적: 전체 구조를 보호
뮤텍스 테이블락

함수 데이터삽입(키: 문자열, 값: 숫자) → 공집합 {
  테이블락.잠금()
  // 전체 테이블 보호 (다른 작업 블로킹)
  테이블[키] = 값
  테이블락.해제()
}

// ✅ 최적화: 버킷별 lock
배열 버킷락들 = [뮤텍스(), 뮤텍스(), ...]

함수 데이터삽입(키: 문자열, 값: 숫자) → 공집합 {
  변수 버킷 = 해시(키) % 길이(버킷락들)
  버킷락들[버킷].잠금()
  // 특정 버킷만 보호 (다른 버킷은 병렬 접근 가능)
  테이블[키] = 값
  버킷락들[버킷].해제()
}
```

### 3. Read-Write Lock

읽기 작업은 병렬, 쓰기는 배타적으로 처리합니다.

```freelang
// 읽기 작업은 동시 실행 가능
읽기락.잠금()
데이터 = 조회(키)
읽기락.해제()

// 쓰기 작업은 독점적
쓰기락.잠금()
데이터[키] = 값
쓰기락.해제()
```

## 실전 예제

### 예제 1: 소수 판정 최적화

```freelang
// ❌ 비효율적: 모든 수로 나누어보기
함수 소수인가_나쁨(n: 숫자) → 참거짓 {
  만약 n < 2 { 반환 거짓 }

  반복 변수 i = 2; i < n; i = i + 1 {
    만약 n % i == 0 { 반환 거짓 }
  }

  반환 참
}
// 시간복잡도: O(n)

// ✅ 최적화: 제곱근까지만 확인
함수 소수인가_좋음(n: 숫자) → 참거짓 {
  만약 n < 2 { 반환 거짓 }
  만약 n == 2 { 반환 참 }
  만약 n % 2 == 0 { 반환 거짓 }

  변수 제곱근 = 정수제곱근(n)

  반복 변수 i = 3; i <= 제곱근; i = i + 2 {
    만약 n % i == 0 { 반환 거짓 }
  }

  반환 참
}
// 시간복잡도: O(√n)
// n=1,000,000: 1,000,000 vs 1,000회 비교
```

### 예제 2: 배열 합계 최적화

```freelang
// ❌ 비효율적: 모듈로 연산
함수 합계_나쁨(배열: 배열<숫자>) → 숫자 {
  변수 합 = 0

  반복 변수 i = 0; i < 길이(배열); i = i + 1 {
    합 = (합 + 배열[i]) % 1000000000  // 불필요한 모듈로
  }

  반환 합
}

// ✅ 최적화: 모듈로 제거 (마지막에만)
함수 합계_좋음(배열: 배열<숫자>) → 숫자 {
  변수 합 = 0

  반복 변수 i = 0; i < 길이(배열); i = i + 1 {
    합 = 합 + 배열[i]
  }

  반환 합 % 1000000000  // 마지막에만 모듈로
}
// 성능 개선: 100배 이상
```

## 성능 벤치마킹

### 1. 벤치마크 작성

```freelang
함수 벤치마크(함수명: 문자열, 함수: 함수, 반복: 숫자) → 맵<문자열, 숫자> {
  변수 시작시간 = 현재시간()

  반복 변수 i = 0; i < 반복; i = i + 1 {
    함수()
  }

  변수 종료시간 = 현재시간()
  변수 소요시간 = 종료시간 - 시작시간

  반환 {
    함수명: 함수명,
    반복횟수: 반복,
    소요시간: 소요시간,
    평균: 소요시간 / 반복
  }
}

// 사용
변수 결과1 = 벤치마크("소수_나쁨", 소수인가_나쁨(1000), 10000)
변수 결과2 = 벤치마크("소수_좋음", 소수인가_좋음(1000), 10000)

출력("나쁨: " + 결과1.소요시간 + "ms")
출력("좋음: " + 결과2.소요시간 + "ms")
출력("개선: " + (결과1.소요시간 / 결과2.소요시간) + "배")
```
