## 1️⃣ 해싱 (Hashing)

### 개념

해싱은 **단방향 암호화**입니다. 원본 데이터에서 고정 길이의 해시값을 생성합니다.

```
평문 "password123"
  ↓ (SHA-256)
해시 "ef92b778bafe773f80cd4fba13f64f8d4130f4db..."
     (되돌릴 수 없음!)
```

### SHA-256 해싱

```freelang
사용 crypto.hash

// 텍스트 해싱
변수 암호 = "비밀번호123"
변수 해시값 = SHA256_해시(암호)
출력(해시값)  // 64자 16진수 문자열

// 파일 무결성 검증
변수 파일내용 = 파일읽기("important.txt")
변수 파일해시 = SHA256_해시(파일내용)
출력("파일 해시: " + 파일해시)
```

### HMAC (서명)

HMAC은 **비밀키가 필요한 해싱**입니다. 메시지 무결성 + 인증을 제공합니다.

```freelang
사용 crypto.hash

변수 메시지 = "중요한 메시지"
변수 비밀키 = "shared-secret-key"

// HMAC 생성
변수 서명 = HMAC_SHA256(메시지, 비밀키)

// 서명 검증
변수 유효 = HMAC_SHA256_검증(메시지, 서명, 비밀키)
출력(유효)  // true
```

### 실전 예제: API 서명

```freelang
사용 crypto.hash

함수 API요청_서명(메서드: 문자열, 경로: 문자열, 본문: 문자열) → 맵<문자열, 문자열> {
  변수 API키 = "app-secret-key"
  변수 타임스탐프 = 현재타임스탐프()

  // 서명할 데이터: 메서드 + 경로 + 타임스탐프
  변수 서명데이터 = 메서드 + "|" + 경로 + "|" + 타임스탐프
  변수 서명 = HMAC_SHA256(서명데이터, API키)

  반환 {
    "X-Signature": 서명,
    "X-Timestamp": 타임스탐프,
    "Authorization": "Bearer " + API키
  }
}
```

## 2️⃣ 암호화 (Encryption)

### 개념

암호화는 **양방향** 처리입니다. 키를 사용하여 복호화할 수 있습니다.

```
평문 "비밀 정보"
  ↓ (AES-256 암호화, 키: "32바이트키")
암호문 "a7f3c8d2e1f9b4c6..."
  ↓ (AES-256 복호화, 키: "32바이트키")
평문 "비밀 정보"
```

### AES-256-CBC 암호화 (권장)

CBC 모드는 각 블록을 이전 블록과 연결하여 더 안전합니다.

```freelang
사용 crypto.aes

// 키 생성 (32바이트 = 256비트)
변수 암호화키 = 임의키_생성()  // "WnR/UkZO8KL..."
출력("암호화 키: " + 암호화키)

// 평문 암호화
변수 평문 = "개인정보: 김프리랭, 010-1234-5678"
변수 암호문 = AES256_CBC_암호화(평문, 암호화키)
출력("암호문: " + 암호문)

// 복호화
변수 복호화됨 = AES256_CBC_복호화(암호문, 암호화키)
출력("복호화: " + 복호화됨)  // "개인정보: ..."
```

### 실전 예제: 데이터베이스 암호화

```freelang
사용 crypto.aes
사용 crypto.hash
사용 io

함수 사용자정보_저장(이름: 문자열, 전화: 문자열) → 공집합 {
  // 마스터키 로드
  변수 마스터키 = 파일읽기("keys/master.key")

  // 민감한 데이터 암호화
  변수 암호화된전화 = AES256_CBC_암호화(전화, 마스터키)
  변수 해시된이름 = SHA256_해시(이름)  // 검색용

  // 저장
  변수 레코드 = "사용자ID|" + 해시된이름 + "|" + 암호화된전화
  파일추가("users.db", 레코드 + "\n")
}

함수 사용자정보_조회(이름: 문자열) → 문자열 {
  변수 마스터키 = 파일읽기("keys/master.key")
  변수 해시 = SHA256_해시(이름)

  // 파일에서 조회
  변수 내용 = 파일읽기("users.db")

  // 간단한 검색 (실제로는 파싱 필요)
  반환 암호화된전화
}
```

## 3️⃣ 일회용 암호 (OTP)

### TOTP (시간 기반)

Google Authenticator, Microsoft Authenticator가 사용하는 방식입니다.

```freelang
사용 crypto.otp

// 1단계: 비밀키 생성 (사용자에게 보여줌)
변수 비밀키 = 비밀키_생성()
출력("비밀키: " + 비밀키)
출력("QR 코드: " + TOTP_QR코드URL(비밀키, "user@example.com", "K-FreeLang"))

// 2단계: 사용자가 Google Authenticator에 비밀키 등록
// (사용자가 앱에서 QR 코드 스캔)

// 3단계: 사용자가 생성한 코드 검증
변수 사용자코드 = "123456"  // 사용자가 앱에서 확인
만약 TOTP_검증(비밀키, 사용자코드) {
  출력("2FA 인증 성공!")
} 아니면 {
  출력("2FA 인증 실패!")
}
```

### HOTP (카운터 기반)

보안 토큰이나 한 번에 하나씩 생성되는 코드에 사용됩니다.

```freelang
사용 crypto.otp

변수 비밀키 = 비밀키_생성()
변수 카운터 = 0

// 로그인 시도
함수 로그인_검증(사용자코드: 문자열) → 참거짓 {
  만약 HOTP_검증(비밀키, 사용자코드, 카운터) {
    카운터 = 카운터 + 1  // 다음 코드를 위해 증가
    반환 참
  }
  반환 거짓
}
```

## 4️⃣ 인코딩 (Encoding)

### Base64 (API 전송)

Base64는 바이너리 데이터를 텍스트로 안전하게 전송할 수 있게 합니다.

```freelang
사용 crypto.encoding

// Base64 인코딩
변수 원본 = "Hello, K-FreeLang!"
변수 인코딩됨 = Base64_인코딩(원본)
출력(인코딩됨)  // "SGVsbG8sIEstRnJlZUxhbmchIQ=="

// Base64 디코딩
변수 복호화됨 = Base64_디코딩(인코딩됨)
출력(복호화됨)  // "Hello, K-FreeLang!"
```

### Hex (디버깅)

16진수 표현은 디버깅과 검증에 유용합니다.

```freelang
사용 crypto.encoding

변수 데이터 = "ABC"
변수 Hex = Hex_인코딩(데이터)
출력(Hex)  // "414243"

변수 복원 = Hex_디코딩(Hex)
출력(복원)  // "ABC"
```

### URL-safe Base64 (JWT, OAuth)

JWT 토큰과 OAuth에서 사용합니다.

```freelang
사용 crypto.encoding

변수 페이로드 = "{\"user_id\": 123, \"role\": \"admin\"}"
변수 토큰 = URLSafeBase64_인코딩(페이로드)
출력(토큰)  // "eyJ1c2VyX2lkIjogMTIzLCAi..." (패딩 없음)
```

## 5️⃣ 통합 예제: 사용자 인증 시스템

```freelang
사용 crypto.hash
사용 crypto.aes
사용 crypto.otp
사용 io

// 글로벌 설정
상수 마스터키 = "32바이트-길이-암호화-키-!!!!"
상수 API키 = "secret-api-key-123"

함수 사용자계정_생성(이메일: 문자열, 암호: 문자열) → 맵<문자열, 문자열> {
  // 1. 암호 해싱 (장기 보관용)
  변수 해시된암호 = 암호_해시(암호)

  // 2. 2FA 비밀키 생성
  변수 OTP비밀키 = 비밀키_생성()

  // 3. 사용자 레코드 저장
  변수 레코드 = 이메일 + "|" + 해시된암호 + "|" + OTP비밀키
  파일추가("accounts.db", 레코드 + "\n")

  반환 {
    "email": 이메일,
    "2fa_secret": OTP비밀키,
    "2fa_qr": TOTP_QR코드URL(OTP비밀키, 이메일, "K-FreeLang")
  }
}

함수 로그인(이메일: 문자열, 암호: 문자열, OTP코드: 문자열) → 참거짓 {
  // 1. 계정 조회 (간단한 예)
  // 실제로는 이메일로 레코드 검색
  변수 저장된암호해시 = "..."
  변수 OTP비밀키 = "..."

  // 2. 암호 검증
  만약 !암호_검증(암호, 저장된암호해시) {
    반환 거짓
  }

  // 3. OTP 검증
  만약 !TOTP_검증(OTP비밀키, OTP코드) {
    반환 거짓
  }

  // 4. API 토큰 생성 (HMAC 서명)
  변수 토큰데이터 = 이메일 + "|" + 현재타임스탐프()
  변수 토큰서명 = HMAC_SHA256(토큰데이터, API키)

  // 5. 토큰 저장 (암호화)
  변수 암호화된토큰 = AES256_CBC_암호화(토큰서명, 마스터키)
  파일추가("sessions.db", 이메일 + "|" + 암호화된토큰 + "\n")

  반환 참
}
```

## 📝 연습 문제

### 문제 1: 데이터 마스킹

민감한 정보(전화번호, 이메일)를 마스킹합니다.

```freelang
함수 전화마스킹(전화: 문자열) → 문자열 {
  // 예: "010-1234-5678" → "010-****-5678"
  // 힌트: string 모듈의 슬라이스 활용
}

함수 이메일마스킹(이메일: 문자열) → 문자열 {
  // 예: "kim@example.com" → "k**@example.com"
}
```

### 문제 2: 암호 강도 검증

암호가 안전한 수준인지 확인합니다.

```freelang
함수 암호강도검증(암호: 문자열) → 참거짓 {
  // 요구사항:
  // - 최소 8자
  // - 대문자 1개
  // - 소문자 1개
  // - 숫자 1개
  // - 특수문자 1개
}
```

### 문제 3: 안전한 토큰 생성

예측 불가능한 세션 토큰을 생성합니다.

```freelang
함수 세션토큰_생성() → 문자열 {
  // 임의키로 시드 생성 후 Base64 인코딩
}
```
