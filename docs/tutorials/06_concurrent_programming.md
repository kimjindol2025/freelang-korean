## 목표

K-FreeLang의 **고급 동시성 구조**를 배웁니다.

- 뮤텍스 (Mutex) & 잠금 (Lock)

- 선택 (Select) - 다중 채널 연산

## 1️⃣ 뮤텍스와 잠금

### 기본 개념

뮤텍스는 **상호 배제** 메커니즘으로, 한 번에 하나의 고루틴만 임계 영역에 진입합니다.

```freelang
// 뮤텍스 선언
뮤텍스 카운터 {
  숫자 = 0
}

함수 안전한증가() → 공집합 {
  // 잠금 획득
  카운터.잠금()

  // 임계 영역: 한 번에 하나만 실행
  변수 현재 = 카운터.값
  카운터.값 = 현재 + 1

  // 잠금 해제
  카운터.해제()
}
```

### RAII 패턴 (Scoped Lock)

자동으로 잠금/해제하는 방식:

```freelang
함수 안전한증가_자동() → 공집합 {
  // 블록 진입 시 자동 잠금, 퇴출 시 자동 해제
  잠금(카운터) {
    변수 현재 = 카운터.값
    카운터.값 = 현재 + 1
  }
}
```

### 실전 예제: 쓰레드 안전 카운터

```freelang
구조체 안전한카운터 {
  뮤텍스 값: 숫자,
}

함수 카운터_생성() → 안전한카운터 {
  반환 {
    값: 0
  }
}

함수 카운터_증가(c: 안전한카운터) → 공집합 {
  잠금(c.값) {
    c.값.값 = c.값.값 + 1
  }
}

함수 카운터_읽기(c: 안전한카운터) → 숫자 {
  잠금(c.값) {
    반환 c.값.값
  }
}

함수 병렬증가() → 공집합 {
  변수 카운터 = 카운터_생성()

  동시실행 {
    반복 변수 i = 0; i < 100; i = i + 1 {
      카운터_증가(카운터)
    }

    반복 변수 i = 0; i < 100; i = i + 1 {
      카운터_증가(카운터)
    }
  }

  출력("최종값: " + 카운터_읽기(카운터))  // 200
}
```

## 2️⃣ 세마포어

### 기본 개념

세마포어는 **신호 메커니즘**으로, 정해진 수의 고루틴만 진입 가능합니다.

```freelang
// 한 번에 3개까지 진입 가능
세마포어 동시실행제한 (3)

함수 작업() → 공집합 {
  동시실행제한.대기()  // 신호 감소

  // 임계 영역: 최대 3개 동시 실행
  출력("작업 시작")
  지연(1000)
  출력("작업 완료")

  동시실행제한.신호()  // 신호 증가
}
```

### 실전 예제: 동시성 제한

```freelang
세마포어 풀 (5)  // 최대 5개 동시 작업

함수 API호출(ID: 숫자) → 공집합 {
  풀.대기()  // 슬롯 획득

  오류처리 {
    변수 응답 = HTTP_GET("/api/" + ID)
    출력("응답 " + ID + ": " + 응답)
  } 잡기 오류 {
    출력("오류 " + ID + ": " + 오류)
  }

  풀.신호()  // 슬롯 반환
}

함수 병렬요청(ID들: 배열<숫자>) → 공집합 {
  동시실행 {
    반복 변수 i = 0; i < 길이(ID들); i = i + 1 {
      API호출(ID들[i])
    }
  }
}
```

## 3️⃣ 채널

### 기본 개념

채널은 **고루틴 간 통신**을 위한 파이프입니다.

```freelang
// 문자열을 전송하는 채널 (버퍼 크기 1)
채널 메시지<문자열>(1)

함수 발신자() → 공집합 {
  메시지 <- "안녕하세요"  // 전송
  메시지 <- "반갑습니다"
}

함수 수신자() → 공집합 {
  변수 msg1 = <- 메시지  // 수신
  변수 msg2 = <- 메시지

  출력(msg1)  // "안녕하세요"
  출력(msg2)  // "반갑습니다"
}

함수 main() → 공집합 {
  동시실행 {
    발신자()
    수신자()
  }
}
```

### 채널 방향 (방향성)

```freelang
// 전송만 가능
채널 송신<숫자>

함수 송신자() → 공집합 {
  송신 <- 42
  송신 <- 100
}

// 수신만 가능
채널 수신<숫자>

함수 수신자() → 공집합 {
  변수 x = <- 수신
  변수 y = <- 수신
}
```

### 버퍼된 채널

```freelang
// 버퍼 크기 0 (동기 채널)
채널 동기<숫자>(0)

// 버퍼 크기 10 (비동기 채널)
채널 비동기<숫자>(10)

함수 버퍼테스트() → 공집합 {
  동시실행 {
    함수 생산자() → 공집합 {
      반복 변수 i = 1; i <= 10; i = i + 1 {
        비동기 <- i  // 10개까지는 블로킹 없음
      }
    }

    함수 소비자() → 공집합 {
      반복 변수 i = 1; i <= 10; i = i + 1 {
        변수 값 = <- 비동기
        출력(값)
      }
    }

    생산자()
    소비자()
  }
}
```

## 4️⃣ 선택 (Select)

### 기본 개념

선택은 **여러 채널 연산 중 하나가 준비될 때까지 대기**합니다.

```freelang
채널 요청<숫자>(5)
채널 응답<문자열>(5)
채널 종료<참거짓>(1)

함수 서버() → 공집합 {
  선택 {
    경우 <- 요청 -> {
      변수 ID = <- 요청
      출력("요청 처리: " + ID)
      응답 <- "처리됨"
    }

    경우 <- 종료 -> {
      변수 신호 = <- 종료
      출력("서버 종료")
    }

    경우기본 -> {
      출력("대기 중...")
    }
  }
}
```

### 타임아웃 구현

```freelang
채널 타이머<참거짓>

함수 응답대기_타임아웃(초: 숫자) → 참거짓 {
  // 별도 고루틴에서 타이머 시작
  동시실행 {
    지연(초 * 1000)
    타이머 <- 참
  }

  선택 {
    경우 <- 응답 -> {
      출력("응답 받음")
      반환 참
    }

    경우 <- 타이머 -> {
      출력("타임아웃!")
      반환 거짓
    }
  }
}
```

## 5️⃣ 실전 예제: 웹 서버 요청 큐

```freelang
구조체 요청 {
  ID: 숫자,
  데이터: 문자열,
}

구조체 응답 {
  ID: 숫자,
  상태: 문자열,
  결과: 문자열,
}

채널 요청큐<요청>(100)
채널 응답큐<응답>(100)

// 워커 풀
함수 워커(ID: 숫자) → 공집합 {
  반복 {
    변수 req = <- 요청큐

    출력("워커 " + ID + ": 요청 " + req.ID + " 처리 중")
    지연(1000)

    응답큐 <- {
      ID: req.ID,
      상태: "완료",
      결과: "처리됨: " + req.데이터
    }
  }
}

// 디스패처
함수 디스패처() → 공집합 {
  반복 변수 i = 1; i <= 10; i = i + 1 {
    요청큐 <- {
      ID: i,
      데이터: "요청 " + i
    }
  }
}

// 응답 수집기
함수 응답수집기() → 공집합 {
  반복 변수 i = 1; i <= 10; i = i + 1 {
    변수 resp = <- 응답큐
    출력("응답 " + resp.ID + ": " + resp.결과)
  }
}

함수 main() → 공집합 {
  동시실행 {
    // 5개 워커 시작
    반복 변수 i = 1; i <= 5; i = i + 1 {
      워커(i)
    }

    // 디스패처 시작
    디스패처()

    // 응답 수집기 시작
    응답수집기()
  }
}
```
